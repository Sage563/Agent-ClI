Security Model

Safety is the primary inhibitor to broader AI adoption. Agent CLI addresses this with a multi-layered security model designed to give you peace of mind.

Sandbox-First Architecture: The agent operates in a restricted environment. It has zero authority to perform destructive system actions (like formatting a drive or deleting your entire home directory) by default.

1. Permission Policies

We use a per-session permission model. You can adjust the strictness level at any time using the /access command.

  • Read-Only: The agent can analyze your code but cannot modify a single character.
  • Selective Access: The agent must request access to specific directories or files.
  • Full Transparency: For trusted environments, allows the agent to modify files within the current workspace project root.

2. Data Privacy & Local Inference

Agent CLI is the only professional-grade agent that defaults to local inference via Ollama. This means your proprietary source code never leaves your local network. For users requiring remote models, we enforce strict SSL/TLS encryption for all API traffic.

3. Command Verification

Unlike other CLI agents that might run shell commands in the background silently, Agent CLI explicitly stages all shell commands in a dedicated TUI panel. In "Ask" mode, it waits for a physical keyboard key-press (Y/N) before executing any external program.

4. Secure Configuration Storage

API keys and sensitive credentials are never stored in plain text in the workspace. They live in a globally-secured agent.config.json file in your user's platform-specific application data directory, with support for environment variable passthrough for CI/CD safety.
