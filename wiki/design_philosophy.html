Design Philosophy

Agent CLI is built on a "Verify then Act" principle. This section details the core values that drive the development of the project.

Transparency First

We believe an AI agent should never be a "black box." Every thought process, tool call, and proposed change is streamed in real-time. This allows developers to understand why the agent is making certain decisions.

Explicit Autonomy

Autonomy is a gradient. Agent CLI allows you to define exactly how much control you want to delegate. From "Prompt Mode" (asking for every change) to "Mission Mode" (full autonomous loop), the power remains with the user.

Tool-Centric Design

The agent is only as powerful as its tools. By embracing the Model Context Protocol (MCP), we prioritize a standard-based expansion system that allows anyone to add capabilities without touching the core logic.

The Code-First Mentality

Unlike general-purpose assistants, Agent CLI is "opinionated" about code. It understands the importance of build systems, version control, and type safety. It's designed to live in your terminal, right next to your compiler.
