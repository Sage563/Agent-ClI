Initial Setup

After installation, follow these steps to configure your environment and connect to an AI provider.

Provider Configuration

Agent CLI supports multiple AI providers. You must choose one and set it as active.

/provider ollama    # Use local models
/provider openai    # Use OpenAI API
/provider gemini    # Use Google Gemini API

Authentication via API Keys

For remote providers, secure your session by adding your API keys to the config:

/config openai_api_key YOUR_KEY_HERE
/config gemini_api_key YOUR_KEY_HERE

Environment Variables (.env)

Alternatively, you can create a .env file in the project root. Agent CLI will automatically sync these values into its internal configuration on startup if env_bridge_enabled is set to true.

OPENAI_API_KEY=sk-...
GEMINI_API_KEY=...
ANTHROPIC_API_KEY=...
OLLAMA_ENDPOINT=http://localhost:11434

Verifying Your Setup

Once you've configured your provider, run the status command to verify the connection:

/status

If the connection is successful, you'll see your current model, provider, and session budget details. If not, refer to the Troubleshooting Guide.

Project-Specific Setup (.agents.json)

For more advanced projects, you can define custom instructions and tools that are specific to a single repository. Create an .agents.json file in your project root to override global settings.
