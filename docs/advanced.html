Advanced Documentation

Take your Agent CLI experience to the next level with advanced scripting, automated workflows, and complex mission planning.

Mission Management

Missions are the most powerful feature of Agent CLI. A mission is a multi-turn autonomous project execution.

Defining a Mission

You can start a mission with a broad goal. The agent will then break it down into phases.

agent-cli --mission "Refactor the entire auth system to use JWT instead of sessions"

Phase Control

The agent will pause after each major phase to let you review progress. You can use the following commands to control the mission:

  • /mission status - See the current phase and progress.
  • /mission pause - Temporarily stop the autonomous loop.
  • /mission resume - Continue with the next planned step.
  • /mission skip - Move to the next phase without finishing the current one.

Custom Tool Injection

While MCP is the preferred way to add tools, you can also inject custom scripts directly into the agent's runtime environment.

The .agents.json File

Place an .agents.json file in your project root to define project-specific tools and instructions.

{
  "project_name": "My App",
  "custom_tools": [
    {
      "name": "deploy_staging",
      "command": "npm run deploy:staging",
      "description": "Deploys the current project state to the staging server."
    }
  ]
}

Automated Testing Integration

You can configure the agent to automatically run tests after every file change. This is critical for maintaining stability during large refactors.

/config auto_verify true
/config verify_command "npm test"

Prompt Engineering

The agent's behavior is driven by its internal system prompt. Advanced users can override or append to this prompt via the agent.prompt.txt file in the config directory.

Instruction Hierarchy

  1. System Default: Core agentic behavior.
  2. Environment Overrides: OS-specific details.
  3. Project Instructions: Found in .agents.json.
  4. User session prompts: Provided via `/prompt` or initial input.

CLI Flags Reference

Flag Description
--verbose Show detailed debugging logs in the terminal.
--no-stream Disable streaming output (good for CI environments).
--config <path> Use a custom configuration file path.
--mission <goal> Initialize the agent with a long-running mission.
--safe-mode Force the agent to ask for permission for EVERY command.
