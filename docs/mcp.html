Model Context Protocol (MCP)

MCP is a revolutionary standard that allows Agent CLI to interact with the world beyond your file system. It connects the agent to a vibrant ecosystem of tools, resources, and templates.

Why MCP?

Without MCP, an AI agent is a "brain in a box," limited to the tools the developers hard-coded. With MCP, you can plug in any capability. Need the agent to read your Google Calendar? Plug in the Calendar MCP server. Need it to query your Postgres database? Use the Database MCP server.

Core Concepts

  • Servers: Independent programs that provide capabilities. They run locally or as microservices.
  • Tools: Executable functions the AI can call (e.g., `get_weather`, `execute_query`).
  • Resources: Static or dynamic data the AI can read (e.g., `schema.sql`, `vignette.md`).
  • Prompts: Predefined instruction templates that guide the AI's behavior for specific tasks.

Managing Servers

Adding a Server

Servers are added via the `/mcp add` command. You can use global packages via npx or uvx.

/mcp add filesystem npx -y @modelcontextprotocol/server-filesystem C:/Users/Docs
/mcp add memory npx -y @modelcontextprotocol/server-memory

Listing Installed Servers

Verify which servers are currently active in your session:

/mcp list

Removing a Server

/mcp remove memory

Using Tools

However, you can list all available tools to see what the agent is capable of:

/mcp tools

Advanced Configuration

You can define default MCP servers that load every time you start the agent by editing your global agent.config.json:

{
  "mcp": {
    "enabled": true,
    "servers": {
      "github": {
        "command": "npx",
        "args": ["-y", "@modelcontextprotocol/server-github"],
        "env": { "GITHUB_PERSONAL_ACCESS_TOKEN": "..." }
      }
    }
  }
}

Developing Your Own MCP Server

Agent CLI uses the standard JSON-RPC over Standard Error/Output for local transport. You can write a server in any language (TypeScript, Python, etc.) and connect it to Agent CLI.

  1. Implement the MCP Protocol: Follow the official specification from modelcontextprotocol.io.
  2. Expose your tools: Define the `tools/list` and `tools/call` methods.
  3. Test locally: Connect it using the `/mcp add` command with the absolute path to your executable.
