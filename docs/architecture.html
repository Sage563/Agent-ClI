System Architecture

A deep dive into the inner workings of Agent CLI. Understanding the architecture is key to extending the agent or contributing to the core.

Overview

Agent CLI is built on a decoupled, service-oriented architecture written entirely in TypeScript. It prioritizes safety, transparency, and extensibility.

The Core Loop

The agent operates in a continuous "Sense-Plan-Act" loop:

  1. Sense: The Task Builder gathers project context, including file contents, directory structures, and recent logs.
  2. Plan: The LLM processes the context and user request to generate a structured `TaskPayload` including proposed `changes[]` and `commands[]`.
  3. Act: The Execution Engine validates the instructions, applies file changes via the Applier, and runs shell commands.

Component Breakdown

1. The Orchestrator (src/core/agent.ts)

The "brain" of the application. It manages session state, handles the dialogue loop, and coordinates between the TUI and the AI providers. It also enforces the session's security policy.

2. Task Builder (src/task_builder.ts)

A sophisticated context-gathering engine. It doesn't just send raw strings; it intelligently selects relevant files, truncates long outputs, and formats the project state into a structure the AI can easily parse.

3. File Applier (src/applier.ts)

The safety layer for file modifications. Instead of simple overwrites, the Applier uses a diff-based approach to ensure that code is inserted correctly. It also maintains an undo-stack, allowing the user to revert changes instantly.

4. Provider Framework (src/providers/)

A standardized interface for different AI backends. Whether you're using Ollama's local API, OpenAI's streaming REST API, or Gemini's Vertex AI, the core agent interacts with them through a consistent `AIProvider` interface.

5. MCP Connector (src/core/mcp.ts)

The bridge to the Model Context Protocol. It dynamically loads MCP servers, translates their JSON-RPC methods into agent tools, and injects their capabilities directly into the AI's prompt.

Data Flow Diagram

[User Input] → [Agent Core] → [Task Builder] → [AI Provider]
                                                    ↓
[Undo Stack] ← [Applier] ← [Execution Engine] ← [AI Response]

Security Philosophy

The architecture is "Sandbox-First". The agent never has write access to your system unless explicitly granted via the configuration or a session-level `/access` command. All file writes are staged and can be reviewed before commitment.
