Complete Command Reference

A comprehensive list of all interactive commands available within the Agent CLI shell.

Command Arguments Summary
/help - Display the interactive help menu.
/provider <id> Switch to a specific AI backend.
/model <tag> Select a specific model version.
/config [key] [v] Manage global agent settings.
/access <mode> Toggle session security permissions.
/mission <goal> Start an autonomous task loop.
/mcp <sub> Interact with Model Context Protocol servers.
/undo - Revert the last batch of file modifications.
/list_diff [n] Review recently applied file changes.
/clear - Wipe the session terminal terminal history.
/status - Show session health and API usage stats.
!<cmd> - Direct shell command execution (shortcut).

Detailed Breakdown

/provider <name>

Changes the current AI backend. Supported values: ollama, openai, anthropic, gemini, deepseek.

/config [key] [value]

Allows real-time modification of the agent.config.json file. Running /config without arguments lists all current settings. Running /config key shows the value for that specific key.

/access <full | selective | prompt>

  • full: The agent can read and write to any file in your project automatically.
  • selective: The agent can only touch files you've explicitly "shared" in the session.
  • prompt: The agent must ask for permission before every file read or write.

/mcp <subcommand> [args]

  • /mcp add <name> <cmd> [args]: Register a new MCP server.
  • /mcp remove <name>: Unregister an MCP server.
  • /mcp list: Show all active servers.
  • /mcp tools: Show all capabilities provided by active servers.
  • /mcp inspect <name>: Show technical details for a server.

/mission <goal_description>

Initializes an autonomous workflow. The agent will ignore some "Ask" policies and attempt to solve the problem by creating a plan and iterating on tool results until the goal is achieved.

/undo

Agent CLI maintains a "transactional" history of file changes. If the AI makes a mistake or you're unhappy with a refactor, /undo will restore your files to their state before the last AI turn.

!<command>

Any input starting with ! is executed directly in your system shell. This is useful for running builders, tests, or git commands without leaving the agent interface.

!npm run build
!git status
