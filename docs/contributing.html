Contributing Guide

Welcome! We're excited that you're interested in contributing to Agent CLI. This project thrives on community input, and we want to make the process as smooth as possible.

Core Technologies

The project is written in TypeScript and runs on Node.js. We use several key libraries to manage the TUI and file operations:

  • fs-extra: For robust file system interactions.
  • path: Core Node.js path management.
  • say: A text-to-speech cross-platform wrapper.
  • mime-types: For identifying file structures automatically.

Development Workflow

1. Set Up Your Environment

Fork the repository and clone it to your local machine. Ensure you have Node.js 20+ installed.

git clone https://github.com/Sage563/Agent-ClI.git
cd Agent-ClI
npm install

2. Run in Development Mode

Use the nodemon-powered dev script to automatically restart the agent when you make changes to the source code.

npm run dev

3. Project Structure

  • src/core/: Core logic, agent state, and orchestration.
  • src/ui/: TUI components, status panels, and output formatting.
  • src/providers/: External AI service integrations.
  • scripts/: Build utilities and runtime asset generation.

Coding Standards

  • TypeScript: Use strong typing whenever possible. Avoid `any` types.
  • Async/Await: Use modern async patterns for all I/O and API calls.
  • Documentation: Document all public classes and functions using JSDoc-style comments.
  • Error Handling: Use try/catch blocks effectively and provide meaningful error messages to the user.

Submitting a Pull Request

  1. Create a feature branch (git checkout -b feature/awesome-new-thing).
  2. Commit your changes with descriptive messages.
  3. Ensure all existing tests pass (npm test).
  4. Push to your fork and submit a Pull Request to our main branch.

Community and Conduct

We believe in a diverse and welcoming community. Please follow our Code of Conduct in all interactions. If you have any questions, feel free to open a Discussion on GitHub or reach out to the maintainers.
