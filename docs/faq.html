FAQ

Answers to common questions about using and configuring Agent CLI.

Does Agent CLI send my code to the internet?

It depends on your provider. If you use Ollama, everything stays local on your machine. If you use OpenAI, Gemini, or Anthropic, the specific files the agent reads are sent to their respective APIs for processing. You can control which files are shared using the /access command.

How do I update Agent CLI?

If you installed via cloning the repo, simply run:

git pull
npm install
npm run build
Can I use Agent CLI with VS Code?

Yes. Many users run Agent CLI in the integrated terminal of VS Code. Changes made by the agent will appear in real-time in your editor.

What is "Mission Mode" versus regular chat?

Regular chat is for one-off questions or small changes. Mission Mode is for broad objectives where the agent can take multiple turns to reach a goal without you having to prompt it for every step.

I'm getting a "Rate Limit" error. What do I do?

This is usually from your AI provider (e.g., OpenAI). You may need to upgrade your API tier or wait for your quota to reset. You can also try switching to a local model with Ollama to avoid limits entirely.

Does it support languages other than TypeScript/JavaScript?

Absolutely. Agent CLI is language-agnostic. It works with Python, Go, Rust, Java, HTML/CSS, and almost any text-based programming language.

How do I reset my configuration?

You can delete the agent.config.json file in your AppData directory, and the agent will regenerate a fresh one with default settings on the next startup.

Can I use custom system prompts?

Yes. You can create an agent.prompt.txt file in your configuration directory to provide custom instructions that the agent will always follow.
