Architecture Overview

Core Principles

The system is designed to be backend-agnostic. The "brain" lives in the Core, while "muscle" operations like file writing and shell execution are isolated in helper modules with strict validation.

Module Structure

  • Core Engine: Manages session state and the dialogue loop.
  • Task Builder: Intelligently assembles project context for the AI.
  • Diff Applier: Safely persists AI-generated code changes.
  • MCP Connector: Bridges the agent to external tools and data via JSON-RPC.
